Page Replacement Algorithms

1. FIFO (First-In First-Out)

FIFO replaces the oldest page in memory first. It functions like a queue, removing the first loaded page.

Steps:

  • Maintain a queue of loaded pages.
  • On page fault, remove the oldest page (front of the queue).
  • Insert the new page at the end of the queue.

Formula:

Page Fault Rate = Page Faults / Total Requests

2. LRU (Least Recently Used)

LRU replaces the page that has not been used for the longest time, assuming it won’t be used soon.

Steps:

  • Track page usage history.
  • Remove the least recently used page on a fault.
  • Add the new page and update usage.

Tip:

Can be implemented using a stack or timestamps.

3. Optimal Page Replacement

This algorithm replaces the page that won’t be used for the longest time in the future — giving the best results theoretically.

Steps:

  • Check all future page references.
  • Replace the page with the farthest next use.
  • Insert the new page in memory.

Note:

Used mostly for comparison since future access is unknown in real-time systems.
